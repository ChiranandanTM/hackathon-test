# Live Attack Demo System Architecture

The Live Attack Demo is a functional on-chain demonstration that contrasts an unprotected wallet drain with an AgentGuard-protected flow.

## 1. Core Architecture

### A. Smart Contracts (Blockchain Layer)
- `contracts/contracts/MockUSDC.sol`
- `contracts/contracts/AttackSimulator.sol`
- `contracts/contracts/AgentGuardOnChain.sol`

### B. Frontend (UI Layer)
- `frontend/src/components/LiveAttackDemo.jsx`
- `frontend/src/agentguard.config.js` (generated by deploy script)

### C. Backend (AI Layer)
- Existing `POST /intercept` endpoint in `backend/app/routes/transactions.py`
- Used by `LiveAttackDemo.jsx` to fetch risk explanation after on-chain execution

## 2. Demo Flow
1. Frontend loads Sepolia config and wallet addresses from `agentguard.config.js`.
2. User starts demo from `Live Attack Demo` tab.
3. Alex unprotected transaction executes: `AttackSimulator.drainWallet(...)`.
4. Sam protected path sends substituted payload to `AgentGuardOnChain.validateAndExecute(...)`.
5. AgentGuard validation reverts on payload substitution.
6. Frontend refreshes balances and renders tx proof links.
7. Frontend calls backend `/intercept` to show simple AI explanation.

## 3. Setup and Run
1. Install dependencies in `contracts/` and `frontend/`.
2. Deploy demo stack:
   - `cd contracts`
   - `npm run deploy:demo`
3. Start backend (for AI explanation endpoint).
4. Start frontend and open `Live Attack Demo` tab.

## 4. Migration Checklist
1. Copy contracts (`MockUSDC`, `AttackSimulator`, `AgentGuardOnChain`).
2. Copy deploy script (`contracts/scripts/deployDemo.js`).
3. Keep generated frontend config pipeline (`frontend/src/agentguard.config.js`).
4. Copy `LiveAttackDemo.jsx` and ensure `ethers` is installed.
5. Expose `/intercept` endpoint in backend for AI explanation.
